# legal_cases

## 一、功能设计

### 1、案例搜索

- 协议：HTTP
- 接口：/cases/search
- 描述：根据前端的筛选逻辑，返回案例基本信息列表

**后端逻辑：**

1. 使用 Redis 缓存，**ZSet** 结构存储，key 为 md5(keyword + country + period)，score 为判决时间，member 为 caseId，超时时间为 24 小时
2. 根据 keyword + country + period 生成 md5 值，查询缓存中是否存在该值
    - 若存在则按分页参数取出 caseId 去数据库中批量查询
    - 若不存在则向下执行
3. 将前端传入的自然语言形式的keyword用**大模型**进行处理，转化为爬虫可接受的关键词，语言与数据源对应
4. 根据国家参数（空、美国、欧盟、日本）调用不同数据源爬虫爬取案例基本信息数据，入参**关键词、时期**，返回案例基本信息列表
5. 将 caseId 和判决时间存储到 Redis ZSet 中 
6. 将案例数据分批保存到数据库（去重）
   - 如果案例已存在（从英文库中查询），不操作
   - 如果案例不存在，先存入英文库（日语结果翻译后存储），再调**百度翻译API**翻译英语为中文，再存入中文库
7. 根据 Redis score 的排序结果，从数据库中取出数据，返回第一页数据给前端
8. 返回前去收藏表中判断案例是否被收藏

### 2、查看案例详细信息

- 协议：HTTP
- 接口：/cases/aisummary
- 描述：根据前端传入的caseId，返回案例详细信息，语言与前端传入的language相同（zh代表中文，en代表英文）

**后端逻辑：**

1. 根据前端传入的caseId，从案例详细信息表中查询是否存在该案例
   - 若存在则直接返回对应语言的案例详细信息
   - 若不存在则向下执行
2. 根据caseId，从案例基本信息表中查询该案例的 url
3. 调用爬虫爬取案例详细信息，入参 url，返回案例详细信息
4. 调用大模型进行总结，并输出 markdown 格式（**英文**）
5. 调用**百度翻译API**翻译为中文，然后存入案例详细信息表
6. 根据前端传入的language，返回对应语言的案例详细信息
7. 查看案例详细信息时，要保存记录到消息浏览历史表中
